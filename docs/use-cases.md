# Сценарии использования

## 1. Быстрое чтение из буфера обмена

**Задача:** Быстро прослушать выделенный текст.

**Шаги:**
1. Выделите текст в любом приложении
2. Скопируйте в буфер обмена (`Ctrl+C`)
3. Нажмите `Ctrl+`` (глобальная горячая клавиша)
4. Текст начнёт воспроизводиться

**Что происходит:**
- Текст извлекается из буфера обмена
- Проходит нормализацию (английские слова → транслитерация, числа → слова)
- Синтезируется аудио через Silero TTS
- Воспроизводится с подсветкой текущего слова

---

## 2. Отложенное чтение (очередь)

**Задача:** Накопить несколько текстов для последовательного прослушивания.

**Шаги:**
1. Скопируйте первый текст
2. Нажмите `Ctrl+Shift+`` — текст добавится в очередь
3. Скопируйте следующий текст
4. Повторите `Ctrl+Shift+``
5. Откройте приложение и нажмите Play

**Полезно для:**
- Чтения документации по частям
- Накопления статей для прослушивания
- Работы с несколькими источниками

---

## 3. Управление воспроизведением

**Горячие клавиши в приложении:**

| Клавиша | Действие |
|---------|----------|
| `Space` | Пауза / Продолжить |
| `←` / `→` | Перемотка ±5 сек |
| `Shift+←` / `Shift+→` | Перемотка ±30 сек |
| `[` / `]` | Уменьшить / Увеличить скорость |
| `N` | Следующий в очереди |
| `P` | Предыдущий в очереди |

---

## 4. Работа с техническим текстом

**Задача:** Прослушать документацию с кодом.

**Пример исходного текста:**
```markdown
## Установка

1. Запустите `pip install package`
2. Версия должна быть >= 2.0.0
3. API доступен на https://api.example.com
```

**Что услышите:**
> Установка.
> Первое. Запустите пип инсталл пэкэдж.
> Второе. Версия должна быть больше или равно два точка ноль точка ноль.
> Третье. Эй пи ай доступен на эйч ти ти пи эс двоеточие слэш слэш api точка example точка ком.

**Обрабатываются:**
- Английские слова → транслитерация (`feature` → "фича")
- Аббревиатуры → по буквам или словом (`API` → "эй пи ай", `JSON` → "джейсон")
- Числа → словами (`123` → "сто двадцать три")
- URL → полное произношение
- Операторы → словами (`>=` → "больше или равно")
- camelCase/snake_case → разделение и транслитерация

## 5. Перегенерация аудио

**Задача:** Обновить аудио после изменения словарей.

**Когда нужно:**
- Добавили новый термин в словарь
- Исправили произношение слова
- Обновили настройки нормализации

**Шаги:**
1. Выберите запись в очереди
2. Нажмите правой кнопкой мыши
3. Выберите "Перегенерировать"
4. Дождитесь завершения синтеза

---

## 6. Просмотр текста с подсветкой

**Задача:** Следить за текстом во время прослушивания.

**Возможности:**
- Текущее слово подсвечивается жёлтым
- Автоматическая прокрутка к текущей позиции
- Переключение между записями в очереди (подсветка только для воспроизводимой)

**Режимы отображения:**
- Markdown — с форматированием (заголовки, списки, код)
- Plain text — без форматирования

---

## 7. Настройка горячих клавиш

**Задача:** Изменить глобальные горячие клавиши.

**Шаги:**
1. Откройте настройки приложения
2. Перейдите в раздел "Горячие клавиши"
3. Установите новые комбинации
4. Сохраните настройки

**По умолчанию:**
- `Ctrl+`` — читать сразу
- `Ctrl+Shift+`` — добавить в очередь

---

## 8. Добавление кастомных терминов

**Задача:** Добавить произношение для нового IT-термина.

**Пример:** Слово "kubernetes" должно читаться как "кубернетис".

**Варианты:**
1. **Через конфиг** — добавить в `custom_it_terms` в настройках
2. **Через код** — добавить в словарь `IT_TERMS` в `english.py`

```python
# В настройках приложения или PipelineConfig
custom_it_terms = {
    "kubectl": "кубектл",
    "nginx": "энжинкс",
}
```

См. [Contributing](contributing.md) для добавления терминов в основной словарь.
